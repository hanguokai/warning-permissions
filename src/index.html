Permissions & Warning Guide

What are permissions and warnings?

See official document Declare permissions and warn users.

In manifest.json, "permissions" and "host_permissions" generate install/update warnings. "optional_permissions" and "optional_host_permissions" do NOT generate warnings on install.

In addition, "content_scripts" also generates host warnings similar to "host_permissions". Special declarations (like overriding the New Tab page) can also trigger warnings.

Important Developer Notes

  • Manifest Version: Set correct "manifest_version" (MV2 vs MV3). Certain permissions are restricted by manifest version.
  • Multiple Hosts Threshold: Declaring 4 or more independent host permissions collapses warnings into: "Read and change your data on a number of websites". Be careful when adding hosts as this may disable extension updates automatically!
  • Tool Scope: Uses Chrome's native chrome.management.getPermissionWarningsByManifest() API to provide exact browser permission warnings without requesting extra permissions.
